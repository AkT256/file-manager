# Лабораторная работа «Файловый менеджер»

## Описание
Проект реализует консольный файловый менеджер на Python. Программа работает только внутри заранее заданной рабочей директории и не позволяет пользователю выйти за её пределы.

## Структура проекта
```text
file_manager_lab/
├── config.py
├── main.py
├── README.md
├── demo.txt
├── file_manager/
│   ├── __init__.py
│   ├── cli.py
│   ├── core.py
│   └── exceptions.py
└── workspace/
```

## Назначение модулей
- `config.py` — настройка рабочей директории.
- `main.py` — точка входа в программу.
- `file_manager/core.py` — основная логика работы с файлами и папками.
- `file_manager/cli.py` — обработка команд пользователя.
- `file_manager/exceptions.py` — пользовательские исключения.

## Команды
- `help` — список команд.
- `pwd` — текущая директория.
- `ls` — список файлов и папок.
- `tree` — дерево каталогов.
- `cd <путь>` — переход в директорию.
- `mkdir <имя>` — создание папки.
- `rmdir <имя>` — удаление пустой папки.
- `touch <файл>` — создание файла.
- `cat <файл>` — чтение файла.
- `write <файл> <текст>` — полная перезапись файла.
- `append <файл> <текст>` — добавление текста в конец файла.
- `rm <файл>` — удаление файла.
- `cp <источник> <назначение>` — копирование файла или папки.
- `mv <источник> <назначение>` — перемещение файла или папки.
- `rename <источник> <новое_имя>` — переименование.
- `info <имя>` — сведения о файле или папке.
- `exit` — выход.
- `zip <что> <архив.zip>` — архивировать.
- `unzip <архив.zip> <куда>` — разархивировать.

## Защита от выхода за пределы workspace
Каждый путь приводится к абсолютному виду, после чего проверяется, что он находится внутри рабочей директории. При попытке обратиться к пути вне `workspace` выбрасывается исключение.
